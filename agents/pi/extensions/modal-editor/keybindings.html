Pi Modal Editor — Key Bindings Draft plan for review — Vim-style modal editing for the pi agent input editor

Context: This is for pi's message input editor — a text area for composing prompts to an AI coding agent. It is not a full file/code editor. There are no files, buffers, windows, LSP, or git to manage — the agent handles those via tools. The keybindings focus on efficient text composition and editing within the input.
Removed from initial plan (not applicable): Buffer management, window/split management, file operations (open/save/rename/recent), project switching, Git integration, Code/LSP actions, toggles (line numbers/minimap/file tree/zen mode), Help/Quit/Session leader submenus (pi has /commands and Ctrl+ shortcuts), marks, macros, gg/G file-level jumps, in-editor search (/, ?, n, N, *, #).

Visual Mode

Enter with v (character) or V (line). Use motions to extend selection, then apply an operator.

Key Action
any motion Extend selection
d / x Delete selection
c / s Change selection (delete + insert)
y Yank selection
> / < Indent / dedent selection
~ Toggle case of selection
u / U Lowercase / uppercase selection
o Swap cursor to other end of selection
Esc Cancel selection → Normal

Numeric Prefix

Most motions and commands accept a count: 3w (3 words forward), 2dd (delete 2 lines), 5x (delete 5 chars).

Leader Key (Normal mode, Space)

A small leader menu for pi-specific actions that benefit from quick access in Normal mode. These are convenience shortcuts — all are also available via /commands or Ctrl+ keys.
Overlay hint: After pressing Space, a popup listing available leader bindings appears after a short delay (default: 500 ms, configurable via setLeaderOverlayDelay(ms)). The overlay dismisses automatically when a binding key is pressed or leader is cancelled.
Key Action Equivalent

Space t Cycle thinking level Shift-Tab
Space o Toggle tool output Ctrl-o
Space k Toggle thinking blocks Ctrl-t
Space e Open in external editor Ctrl-g
Space n New session /new
Space r Resume session /resume
Space b Session tree (branch) /tree
Space c Compact context /compact

Git Sub-menu (Space g)

Leader keys support nested sub-menus. Press Space then g to open the git sub-menu. The overlay updates to show sub-menu bindings, and the pending indicator shows the full path (e.g. SPC g).
Key Action Equivalent
Space g d Git diff /diff

Configuration

Leader menus are defined in keybindings.json alongside the extension source. The file supports multiple leader keys, nested sub-menus, and two action types.

File Format

{
  "menus": {
    "space": {
      "label": "Leader (Space)",
      "key": " ",
      "items": {
        "n": { "label": "New session", "action": "command:/new" },
        "m": { "label": "Switch model", "action": "passthrough:ctrl+l" },
        "g": {
          "label": "+git",
          "items": {
            "d": { "label": "Diff", "action": "command:/diff" }
          }
        }
      }
    }
  }
}

Action Types

Prefix Description Example
command: Submit a slash command command:/new, command:/diff
passthrough: Forward a key combo to pi passthrough:ctrl+l, passthrough:shift+tab

Supported Passthrough Keys

Name Pi Action
ctrl+l Open model selector
ctrl+o Toggle tool output
ctrl+t Toggle thinking blocks
ctrl+g Open external editor
shift+tab Cycle thinking level
Multiple leaders: You can define additional leader keys by adding more entries under menus with different key values (e.g. "," or "\\"). Each menu is independent with its own overlay and sub-menu tree.

Design Principles

  1. Insert-first — Starts in Insert mode since most interactions are typing a prompt and hitting Enter. Normal mode is there when you need to edit, not as the default.
  2. Composability — Operators × motions × text objects. Learn a few building blocks, combine freely.
  3. Passthrough by default — All existing pi Ctrl+ shortcuts work unchanged in every mode. The modal layer only intercepts printable keys in Normal/Visual modes.
  4. Vim grammar, not Vim scope — Core editing grammar (motions, operators, text objects) is preserved, but features that don't apply to a message input (files, buffers, windows, LSP, macros, marks) are omitted.
  5. Counts — Numeric prefixes work with motions and operators (3dw = delete 3 words).
