* Description
Personal additions on top of Spacemacs' built-in =+lang/org= layer:
custom org-mode tweaks, archive helper, web tools, =org-roam= setup,
and the =tasks-org= local package for the plain-org task-memory
protocol.

** Features
  - Hierarchical archive (=org-archive-subtree-hierarchical=)
  - org-web-tools for web capture
  - org-roam (with custom capture templates)
  - =tasks-org= local package (see below)

* Install
To use this configuration layer, add it to your =~/.spacemacs=. You will need to
add =org-user= to the existing =dotspacemacs-configuration-layers= list in this
file.

* tasks-org: plain-org task memory helpers

Local package providing Emacs-side helpers for the plain-org
task-memory protocol used by the =pi= tasks extension.

Files remain plain org. Pi (when present) reloads via its file
watchers; there is no live IPC. The package is useful even without pi
because it just maintains conventions on top of standard org.

** Activation

=tasks-org-mode= auto-enables on any org buffer that matches either rule:

- The buffer contains a =#+DEFAULT_PLAN_DIR:= keyword (canonical marker
  of a task-memory root file — matches =TASKS.org= regardless of where
  it lives relative to the project).
- The buffer's path (relative to the project root) matches a pattern in
  =tasks-org-auto-enable-paths= (defaults: =TASKS.org=, =design/log/*.org=).

Manual activation: =M-x tasks-org-mode=.

Project root is resolved via =projectile-project-root= when projectile
is loaded, falling back to the buffer's directory otherwise.

** Commands & key bindings

All bindings are registered under the org local-leader prefix =, ;=
(=SPC m ;= in Spacemacs). =;= is free in the Spacemacs org layer and
reachable without Shift, with no unbinding gymnastics required.

| Key       | Command                            | Description                                                   |
|-----------+------------------------------------+---------------------------------------------------------------|
| =, ; s=   | =tasks-org-toggle-selected=        | Toggle =#+SELECTED:= on the current task (single-selection)   |
| =, ; p=   | =tasks-org-open-plan=              | Open the =#+IMPORT:='d change-record (errors if absent)       |
| =, ; P=   | =tasks-org-open-plan-other-window= | Same, but open in another window                              |
| =, ; t=   | =tasks-org-jump-to-parent-task=    | Jump to the task in =TASKS.org= whose =#+IMPORT:= is this file   |
| =, ; L p= | =tasks-org-publish-task=           | Publish local task → =TASKS.org= (moves subtree, saves both)  |
| =, ; L u= | =tasks-org-unpublish-task=         | Unpublish top-level task → =TASKS.local.org= (top-level only) |

The mode lighter is =Tasks=.

** Scope

This package is for *review, edit, and reorganisation* of existing
task graphs and change-records. Plan creation and task creation are
the remit of the agent harness (the pi tasks extension's `/tasks new',
=n=, =N=, and =p= keybindings); =tasks-org-open-plan= opens an
existing =#+IMPORT:= linked file but does not scaffold a new one.

** Protocol conventions

See =agents/skills/org-tasks/SKILL.md= for the canonical file
protocol (TODO cycle, properties, =#+IMPORT:=, =#+DEFAULT_PLAN_DIR:=,
=#+SELECTED:=). Emacs-side specifics:

- =:ID:= backfill is the responsibility of the pi tasks extension
  and the agent org-memory skill; this package does not scan or
  mutate IDs, except to assign one when a task is selected via
  =tasks-org-toggle-selected=.

- =#+SELECTED:= is written to the gitignored =TASKS.local.org= as a
  single =#+SELECTED: <UUID>= keyword. Deselecting clears the value
  but retains the file (it may carry local drafts or =#+IMPORT:=
  keywords alongside the selection).

- =#+DEFAULT_PLAN_DIR:= is read from =TASKS.org= and used as an
  auto-enable hint so plan files inside the project's change-record
  directory pick up =tasks-org-mode= automatically.
  =tasks-org-plans-directory= (default: =design/log=) is the
  fallback when the keyword is absent.

** Relationship to the pi tasks extension

Both consumers share the same plain-org files. Pi's file watchers
pick up Emacs saves automatically; the compact widget and overlay
refresh without manual intervention. Plan/task creation lives on
the pi side; Emacs is the reading and refactoring environment.

* Key bindings

See [[#commands--key-bindings][tasks-org commands & key bindings]]
above for the =, ;= prefix. Other layer bindings remain those
inherited from Spacemacs' =+lang/org= layer.
