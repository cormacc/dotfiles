** TODO [#D] Add Nix-level enable flag for individual pi extensions :dotfiles:nix:feature:

Captured during the vim-mode toggle simplification (bfbc6354). Currently agents.nix symlinks the entire `agents/pi/extensions/` directory into `~/.pi/agent/extensions/`, so per-extension disablement requires either:

- Removing the source directory (lossy)
- Manually unlinking the target after `home-manager switch` (fragile)

Better: option-based Nix configuration that materialises only the requested subset of extensions.

Sketch:

```nix
{ piExtensions = {
    enable = [ "tasks" "term" "git-diff" "leader-menu" "vim-mode" ];
    # or: disable = [ "vim-mode" ];
  };
}
```

agents.nix iterates the list and creates per-extension symlinks under `~/.pi/agent/extensions/<name>` rather than one bulk symlink. Probably 30-50 lines of Nix.

Useful for:
- Trying out / disabling vim-mode without removing the source
- Per-host extension subsets (e.g. fewer extensions on the minimal home-manager profile)
- Testing builds with one extension excluded

Final-review finding from the keybindings refactor (parent 00e1772c). Current defaults keep `localLeader` as comma, while vim-mode also adopted Vim repeat-find-backward on `,`.

Observed final state:
- `alt+,` opens the local menu globally.
- In vim-mode Normal mode, bare `,` is handled by the Vim grammar before leader delegation, so the local leader overlay does not open.
- `leader-menu` also warns about `localLeader ","` clashing with the Vim grammar, which means the default config can produce its own warning.

Decide and implement one of:
1. Change the default local leader to a non-conflicting key (candidate `;`, but note Vim `;` is repeat-find-forward).
2. Keep comma as default and document that bare Normal-mode local leader is unavailable; use `alt+,` or configure `leader-menu.json`.
3. Add a priority/escape mechanism so users can choose whether leader or Vim grammar wins for conflicting keys.

Do not resolve accidentally by dropping repeat-find-backward; that was intentionally imported from the upstream evaluation.
