* Summary
Add Open WebUI to the =strix= NixOS profile as the LAN-facing chat UI for the co-hosted Lemonade OpenAI-compatible API. Phase 1 should keep the implementation small: enable the upstream =services.open-webui= NixOS module, keep persistent state under its default =/var/lib/open-webui= state directory, point OpenAI-compatible provider configuration at Lemonade, and expose a single LAN-facing Open WebUI port. Do not enable model serving in Open WebUI; Lemonade remains the only inference backend.

** Decisions
- Use Open WebUI for the initial implementation. It is the best fit for a lightweight local-LLM chat layer over Lemonade and is available as =nixpkgs#open-webui= with an upstream =services.open-webui= NixOS module.
- Do not use LibreChat initially. It is viable but more platform-like and overbuilt for a domestic LAN chat layer.
- Do not use LobeChat initially. It is viable and polished, but current deployment is Docker/full-stack oriented and no =nixpkgs= package or NixOS service was found.
- Design toward the home-network direction in [[file:../home-network-infrastructure.org][home-network-infrastructure.org]] for browser access: eventually =LAN -> Caddy -> Open WebUI -> Lemonade=.
- Keep the raw LAN Lemonade API reachable on =strix=. Coding harness clients on other machines use the OpenAI-compatible API directly, so Open WebUI is an additional chat frontend rather than the only LAN-facing access path.

** Gotchas
- =open-webui= is unfree in nixpkgs, but the repo already enables unfree packages for NixOS builds.
- =services.open-webui= exists. Earlier evaluation missed it because evaluating =nixosConfigurations.strix.options= tripped over the bootstrap placeholder hardware configuration; the nixpkgs source module is at =nixos/modules/services/misc/open-webui.nix=.
- The module exposes =host=, =port=, =stateDir=, =environment=, =environmentFile=, and =openFirewall=. It sets =StateDirectory = "open-webui"= and runs =open-webui serve --host ... --port ...=.
- The module's =services.open-webui.environment= default (=SCARF_NO_ANALYTICS=, =DO_NOT_TRACK=, =ANONYMIZED_TELEMETRY=) is a default *value*, not a base that merges. Any =environment= we set must re-include those telemetry-off entries or they will be silently re-enabled.
- Open WebUI persists configuration to its SQLite DB under =stateDir= after first start. =services.open-webui.environment= entries act as initial values only; later changes are ignored unless we change them via the admin UI or reset state. Plan declarative env around "first-run seed" semantics, not live config.
- The module hard-codes =WEBUI_URL = "http://localhost:${cfg.port}"= in its environment but merges =cfg.environment= after, so we can override =WEBUI_URL= when we move behind Caddy (e.g. =ai.home.arpa=). Not required in phase 1, but expected at phase 2.
- =open-webui= is not in the =nix-amd-ai= Cachix; first build on =strix= will fetch ~301 MiB and build the frontend (npm) and Python deps. Expect a long initial activation.

* Implementation
Relevant background was extracted to [[file:../home-network-infrastructure.org][design/home-network-infrastructure.org]]. Corrected the initial service approach after confirming =services.open-webui= exists in the NixOS module set; use the upstream module rather than hand-writing a local systemd unit.

Implemented in =hosts/strix/nixos-configuration.nix=:
- Added =openWebUIPort = 8080= next to =lemonadePort=.
- Kept Lemonade bound to =0.0.0.0= and kept its firewall hole open because coding harness clients on other LAN machines use the raw OpenAI-compatible API directly.
- Enabled =services.open-webui= with =host = "0.0.0.0"=, =port = openWebUIPort=, and =openFirewall = true=.
- Seeded Open WebUI's OpenAI-compatible provider with =OPENAI_API_BASE_URL = "http://127.0.0.1:13305/v1"= and placeholder =OPENAI_API_KEY = "sk-local-lemonade"=.
- Disabled Ollama integration with =ENABLE_OLLAMA_API = "False"= so Open WebUI remains a frontend, not a second model-serving path.
- Repeated the module's telemetry-off defaults in =services.open-webui.environment= to avoid replacing them accidentally.
- Added =systemd.services.open-webui.after/wants = [ "lemond.service" ]=; the nix-amd-ai module names the Lemonade unit =lemond=.

First-run notes:
- Open WebUI state lives under the module default =/var/lib/open-webui=, with data under =/var/lib/open-webui/data=.
- The first browser sign-up becomes the admin account. After first-run, disable or restrict signup in the Open WebUI admin UI.
- Open WebUI persists provider/auth settings in its SQLite DB after first start. If first-run config is wrong, fix via the admin UI or stop the service and reset =/var/lib/open-webui= before starting again.
- Caddy/SSO remains a phase-2 concern tracked in [[file:../home-network-infrastructure.org][design/home-network-infrastructure.org]]; phase 1 intentionally exposes Open WebUI directly on port 8080.
