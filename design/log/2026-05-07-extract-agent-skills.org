* Summary
Extracted the reusable agent surface (generic skills, pi extensions, prompts, the pi-side =AGENTS.md=, the org-memory task graph and design log) from =~/dotfiles/agents/= into [[https://github.com/cormacc/dotagents]] and consumed it back into dotfiles as a git submodule under =agents-src/=. Dotfiles keeps only the installation wiring (=agents.nix=) and user-local pi configuration (=agents-config/pi/settings.json=). The submodule is symlinked into =~/.agents= and =~/.pi/agent= via =mkOutOfStoreSymlink= so pi's =/reload= still picks up live edits without a Home Manager switch.

** Decisions
- Submodule + =mkOutOfStoreSymlink=, not Nix store copy :: live editability via =/reload= is the daily workflow; staging through the Nix store would defeat that.
- User-local pi config stays in dotfiles :: =agents-config/pi/settings.json= holds machine-specific provider/model/secrets toggles; only neutral defaults live in dotagents.
- pi-settings clean filter ships in dotfiles :: strips volatile fields (=defaultProvider=, =defaultModel=, =lastChangelogVersion=) so =/model= swaps don't churn version control.
- Two-tier global feedback :: skill / extension / prompt feedback routes to dotagents; installation, packaging, dotfiles-side =AGENTS.md= feedback routes to dotfiles. Self-improvement skill rework captured as a follow-up.
- Disabled extensions move as-is but are excluded from packaging :: =pi/extensions/disabled/= preserved as source/history; excluded from =package.json= =pi.extensions= / =files=, the Nix slice, and all flake outputs.
- =pi-clojure= excluded from the org-memory bundle :: Clojure-specific tooling belongs in a separate package (follow-up).

** Shipped
- New repo =github:cormacc/dotagents= containing =skills/=, =pi/extensions/=, =pi/skills/=, prompt templates, =AGENTS.md=, =TASKS.org= + =design/log/=, and the =agent-org-memory= pi package.
- Dotfiles =agents.nix= rewritten to symlink the submodule working tree and run =npm install --omit=dev= on activation for local-only extensions when their =package.json= changes. Activation refuses with an actionable error when the submodule is uninitialised.
- =nix flake check= on dotagents evaluates on every supported system; =agent-org-memory= builds byte-identically from the local submodule and from =github:cormacc/dotagents=.
- Documentation in both repos covers bootstrap, update, editing, release.
- User-local config moved =git mv agents/pi/settings.json agents-config/pi/settings.json=; entire old dotfiles =agents/= tree deleted.

** Gotchas
- Two-step bootstrap on a fresh clone: =git submodule update --init --recursive= + =agents-config/install-git-filter.sh=. Home Manager activation refuses without the first; clean filter requires =jq=.
- Editing skills directly in the live symlink target (=agents-src/=) is the intended workflow; commits land in dotagents, then the dotfiles submodule pointer is bumped in a follow-up commit.
- Top-level Nix =assert= caused infinite recursion during module evaluation when checking submodule presence; the working version is a =home.activation= script wired before =writeBoundary= (per-user =assertions= silently drop under nix-darwin's =home-manager.users.<user>= because they don't propagate to the parent system).
- =emacsclient/test.sh= fails when =tsx= is not on =PATH= — pi bundles =tsx= for runtime but the test wrapper assumes it's reachable. Pre-existing, not introduced by the extraction.
