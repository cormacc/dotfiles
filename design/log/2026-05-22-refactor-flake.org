* Implementation

** nix-caches lib placement

First pass tried =nixConfig = let caches = import ./lib/nix-caches.nix; in { … };=. Nix rejects this with =expected a set but got a thunk= because =nixConfig= is parsed by the lock-file machinery before evaluation — it must be a literal attrset of literals. Reverted that one site to inline lists with a sync-warning comment; the lib still removes the duplication between =nixos-base.nix= and =darwin-configuration.nix=, which are the two paths that drive the running system.

** Dirty-tree gotcha

Nix copies only tracked files into the store when the working tree is dirty. The first =nix eval= attempt failed with =path '…/lib/nix-caches.nix' does not exist= until =git add lib/nix-caches.nix=. Captured as a rule in the new =nix= skill under *Validation*.
