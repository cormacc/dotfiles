  =hosts/t470p/= directory and the legacy single-config
  ~nixosConfigurations.t470p~ branch in =flake.nix=. =t470-nas= goes
  with it (same hardware).
- Defer ~programs.npm.npmrc~ migration until darwin moves off the
  release-25.11 pin; tracked as a follow-up on the existing
  ~nixpkgs#507531~ TODO in =flake.nix=.
- =programs.ssh.matchBlocks= is deprecated on Home Manager *unstable*
  in favour of =programs.ssh.settings= (same DAG-of-attrset shape),
  but darwin HM release-25.11 does **not** yet have =settings= and
  fails with "option does not exist" when it's used. Keep
  =matchBlocks= on home-core.nix until the darwin pin advances past
  the rename; tracked with a comment alongside the existing
  =nixpkgs#507531= follow-up.
- =programs.git.settings.*= IS valid on both pinned HM versions; the
  legacy =userName= / =userEmail= attrs still resolve there but trace
  as obsolete. Use =settings= everywhere; no fallback needed.
- Personal-vs-work git identity convention: default =user.email= =
  =$EMAIL= (work / global), plus =includeIf "gitdir:~/dev/"= scoping
  =$EMAIL_OSS=. =$EMAIL_OSS= falls back to =$EMAIL= when unset so the
  single-identity case is a no-op rather than an empty-string write.

** Gotchas

- =nix.settings.substituters= replaces, =nix.settings.extra-substituters=
  merges. The current mix means an entry added only to ~flake.nix~
  ~nixConfig~ is silently dropped on NixOS hosts.
- =builtins.fetchGit= with no =rev= re-fetches on every evaluation; used
  today for spacemacs, doom, corgi, and the practicalli astro nvim
  config. Pinning a =rev= + =narHash= or moving to ~flake.nix~ inputs
  both fix it.
- =nixos-base.nix= mounts =/mnt/mfs= unconditionally; the =nas= host
  (the moosefs cluster's likely server) inherits the mount.

* Implementation

** nix-caches lib placement

First pass tried =nixConfig = let caches = import ./lib/nix-caches.nix; in { … };=. Nix rejects this with =expected a set but got a thunk= because =nixConfig= is parsed by the lock-file machinery before evaluation — it must be a literal attrset of literals. Reverted that one site to inline lists with a sync-warning comment; the lib still removes the duplication between =nixos-base.nix= and =darwin-configuration.nix=, which are the two paths that drive the running system.

** Dirty-tree gotcha

Nix copies only tracked files into the store when the working tree is dirty. The first =nix eval= attempt failed with =path '…/lib/nix-caches.nix' does not exist= until =git add lib/nix-caches.nix=. Captured as a rule in the new =nix= skill under *Validation*.
