* Summary

Sweep of the dotfiles flake to remove dead hosts, reduce duplicated Nix / HM
configuration, and capture current platform gotchas. All scoped plan tasks are
closed; remaining work is intentionally deferred follow-up, not blocker work.

** Decisions

- Running-system cache config lives in =lib/nix-caches.nix= and is consumed by
  =nixos-base.nix= / =darwin-configuration.nix=. =flake.nix= =nixConfig= stays
  literal because Nix parses it before flake evaluation; keep it manually in
  sync with the lib.
- =strix= keeps only =nix-amd-ai.cachix.org= as host-local
  =extra-substituters= / =extra-trusted-public-keys=. Do not make
  =nix-amd-ai.inputs.nixpkgs= follow this flake's nixpkgs.
- =t470p= and =t470-nas= were dead hardware and are removed.
- =programs.ssh.matchBlocks= remains in =home-core.nix= despite HM unstable's
  deprecation warning: darwin HM release-25.11 rejects the newer
  =programs.ssh.settings= option.
- =programs.git.settings.*= is valid on both current HM pins; legacy
  =userName= / =userEmail= aliases trace as obsolete.
- Git identity convention: =$EMAIL= is the default / work address; repos under
  =~/dev/= use =$EMAIL_OSS= via =includeIf "gitdir:~/dev/"=. =EMAIL_OSS=
  falls back to =$EMAIL= when unset.
- Spacemacs and Practicalli AstroVim intentionally track upstream branch tips
  on install/update. Do not pin their =builtins.fetchGit= calls unless the
  desired behaviour changes to reproducible editor snapshots.

** Shipped

- Added =lib/nix-caches.nix= and wired NixOS + darwin runtime
  =nix.settings.{substituters,trusted-public-keys}= through it.
- Added a reusable top-level =nix= skill under =dotagents/skills/nix=.
- Deleted =hosts/t470p/= and removed =nixosConfigurations.t470p= /
  =t470-nas=.
- Collapsed =nixos-nvidia.nix= / =nixos-nvidia-legacy.nix= into one module with
  =dotfiles.nvidia.legacy= selecting =hardware.nvidia.open= + driver package.
- Replaced SSH string =extraConfig= with named =matchBlocks= for =github.com=,
  =gitlab.com=, =gitlab=, plus the existing =*= defaults. This fixed the
  missing =Host gitlab.com= header.
- Confirmed =programs.git.settings.*= works on linux HM unstable and darwin HM
  release-25.11; switched default git email from hard-coded gmail to =$EMAIL=
  and added the =~/dev/= =$EMAIL_OSS= include.
- Trimmed =nixos-bootstrap.nix= to the installer-ISO delta: flakes,
  bootloader, NetworkManager, stub user, openssh, and git/curl/wget/vim.
- Audited =builtins.fetchGit= calls. Doom is commented out, Corgi is a local
  symlink, and the two live editor fetches now deliberately float on their
  selected branches. Practicalli source URL updated to =practicalli/nvim-astro=.

** Gotchas

- =nix.settings.substituters= replaces; =extra-substituters= merges. Adding a
  cache only to =flake.nix= =nixConfig= does not configure NixOS hosts.
- Dirty flakes only copy tracked files into the Nix store. =git add= new files
  before evaluating config that imports them.
- HM SSH option names differ by pin: =settings= on linux unstable,
  =matchBlocks= on darwin release-25.11.
- =nixos-base.nix= still mounts =/mnt/mfs= unconditionally; =nas= inherits it.

** Follow-ups

- Persisting =EMAIL_OSS= in =home.sessionVariables= would make future impure
  rebuilds from a fresh shell keep the OSS include active. Today bootstrap can
  pass it for the first switch, but a later shell without =EMAIL_OSS= falls
  back to =$EMAIL=.
- =programs.npm.npmrc= migration remains deferred until the darwin pin moves
  past the release-25.11 / =nixpkgs#507531= constraint.
- Onedrive user-unit and Chrome =.app= plist generation were noted in the
  review but left unchanged because the current code works.

* Implementation

- =nixConfig= cannot import =lib/nix-caches.nix= because it is parsed before
  flake evaluation. Runtime Nix settings use the lib; flake-level cache hints
  stay literal with a sync warning.
- New files imported by a dirty flake must be staged before =nix eval=, because
  Nix evaluates the tracked-file snapshot.
- =programs.ssh.settings= worked on linux HM unstable but failed on darwin HM
  release-25.11, so the cross-platform config reverted to =matchBlocks=.
- Editor config fetches were deliberately left floating after review: latest
  upstream branch tip per install/update is preferred over reproducible editor
  snapshots.
