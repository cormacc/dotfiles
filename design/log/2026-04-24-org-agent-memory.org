** DONE [#B] Investigate PLAN as an Emacs hyperlink :org:emacs:
  CLOSED: [2026-04-24 Fri 20:30]
  Findings:
  - Emacs org-mode follows `[[file:...]]` / `[[file:...][desc]]` links in
    property drawer values via `C-c C-o` (`org-open-at-point`). No custom
    link type is required.
  - Extended the parser to detect org-link syntax in the `:PLAN:` value and
    extract the target path, while storing the raw value (`planRaw`) so
    the original form round-trips byte-for-byte.
  - Supported forms:
      :PLAN: plans/foo.org
      :PLAN: [[plans/foo.org]]
      :PLAN: [[file:plans/foo.org]]
      :PLAN: [[file:plans/foo.org][Description]]
  - When the `p` keybinding creates a new plan, it now writes the link form
    (`[[file:...]]`) by default so the resulting property is clickable in
    Emacs without any user action.
  - Verified via the `.agents/tmp/verify-plan-link.ts` smoke test that all
    four forms round-trip cleanly and produce the same extracted path.
