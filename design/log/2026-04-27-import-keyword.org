* Context

The original `:INCLUDE:` mechanism stored the link to an imported task file
inside a task's =:PROPERTIES:= drawer.  This meant it could only ever be
attached to a task heading that had a TODO state — it was impossible to use on
non-task (category) headings like =* Improvements= or at the file root level
(before any heading), which blocked the goal of splitting =TASKS.org= across
multiple files via a lightweight index.

The trigger for this work was the task "Update tasks skill to allow INCLUDE at
top level -- not just for plans", added when the property was still called
=:PLAN:=.  The rename to =:INCLUDE:= (commit =c9ddda9=) addressed the "not just
for plans" semantic aspect but left the "top level" capability unimplemented.

* Design

Replaced =:INCLUDE:= with a =#+IMPORT:= keyword that can appear anywhere in an
org file:

- *In a task body* (after the =:END:= of the properties drawer, before any
  description text) — same behaviour as the old =:INCLUDE:= property but without
  requiring the drawer.
- *At file root level* (before any heading) — tasks from the imported file are
  injected at the root of the task tree.
- *After a category/non-task heading* — =currentTask= is null at that point, so
  the import is also treated as root-level.

The keyword uses org-link syntax for clickability in Emacs (=C-c C-o=):

#+begin_example
#+IMPORT: [[file:design/log/2026-04-25-feature.org]]
#+end_example

Plain-path values are also accepted and round-trip cleanly.

** Trade-offs accepted

- =#+IMPORT:= in a heading body is not a standard org affiliated keyword; Emacs
  displays it as plain text rather than as a drawer property.  Clickability via
  =C-c C-o= on the link value still works.
- =org-entry-get/put= can no longer be used from Emacs Lisp to read/write the
  link; =tasks-org.el= was updated to use regex-based helpers instead.

* Implementation notes

The =fileImports= returned by =parseTasks= for a plan file are resolved one
level deep inside =loadLinkedPlans= (nested =fileImports= within those imported
files are not recursively chased in this iteration).  Cyclic imports are not
detected; avoid them.

The =tasks-org--set-import= elisp helper searches for an existing =#+IMPORT:=
line first (replace-in-place), then falls back to inserting after the last
structural line (=:END:=, =CLOSED:=, or the heading itself).  It does not
remove a =#+IMPORT:= line; use =M-x tasks-org-open-plan= to manage links.
