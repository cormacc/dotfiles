* Context

=TASKS.local.org= is a gitignored per-contributor file.  It currently holds
two pieces of state:

1. =#+SELECTED: <UUID>= — the active task pointer (already implemented).
2. Any number of org task headings — private draft tasks that have not yet
   been promoted to the shared =TASKS.org=.

The extension currently ignores the task headings entirely; =writeSelectedId=
also *overwrites the whole file* on every selection toggle, which would
silently destroy any local tasks the user has written.

This plan covers the full feature:

- Surface local tasks in the extension UI with a clear visual distinction.
- Fix =writeSelectedId= to be non-destructive.
- Publish: promote a local task to =TASKS.org= (makes it shared/tracked).
- Unpublish: demote a shared task to =TASKS.local.org= (removes it from git).
- Emacs-side bindings for publish/unpublish.

** Design decisions

- =isLocal= flag on Task :: A boolean field on the =Task= interface marks
  tasks whose =sourcePath= is =TASKS.local.org=.  This is set by the loader,
  not stored in the org file itself (no tag pollution).

- Visual presentation :: Local tasks render with a =⊠= tree marker instead
  of =•=/=▶=/=▼= and receive a magenta tint (distinct from the accent used for
  selected tasks).  A thin separator line labelled =⊠  Local drafts= is
  injected between the shared task list and the local group.  No separate
  panel; the two groups share the same scrollable left pane.

- Load order :: Local tasks are appended after all shared tasks so the shared
  workstream always appears first.

- =writeSelectedId= non-destructive fix :: Read the existing file, replace or
  insert the =#+SELECTED:= line in-place (regex replace on the content string),
  then write atomically.  Tasks in the file are untouched.

- Save path for local tasks :: =task.sourcePath= is already set to the file
  the task came from.  The existing =writeTaskFilePreserving= flow writes to
  =sourcePath=, so status edits to local tasks automatically go to
  =TASKS.local.org= without any extra routing logic.  The =#+SELECTED:= line
  is non-task content and is preserved verbatim by the preserving serializer.

- Publish (local → shared) :: The task subtree is removed from
  =TASKS.local.org= and appended to =TASKS.org= as a new top-level entry.
  =task.lineNumber= is set to 0 so =serializeTasksPreservingFile= treats it
  as a new root.  =sourcePath= and =sourceRoot= are updated to point at
  =TASKS.org=.  Keybinding: =P= (shift-p) in the overlay.

- Unpublish (shared → local) :: The task is removed from =TASKS.org= and
  appended to =TASKS.local.org=.  Symmetric to publish.  Only allowed for
  top-level tasks (same constraint as archiving).  Keybinding: =U= (shift-u)
  in the overlay.

- Archive guard :: Archiving a local task is refused with a notification
  ("Local tasks cannot be archived — publish first or delete manually").

- New local task :: =n= / =N= keybindings when the cursor is on a local task
  create the new task in =TASKS.local.org= rather than =TASKS.org=.  When
  the cursor is on a shared task, behaviour is unchanged.

- Emacs side :: =tasks-org-publish-task= and =tasks-org-unpublish-task=
  commands in =tasks-org.el=, bound under =, ; L= (local) sub-map.  Both
  operate on the heading at point and perform an atomic move between the two
  files.

* Open questions

1. *Separator row navigability* → render-time injection only.  The row array
   stays clean; the render method detects the shared/local boundary by index
   and injects the divider line without adding a navigable row.

2. *Unpublish scope* → top-level shared tasks only, symmetric with archive.
   Extracting a subtask to local requires a deliberate two-step (unpublish
   the whole parent, rework locally, re-publish what you want).

3. *Local task =#+IMPORT:=* → supported without special-casing.  The parser
   and loader already handle it; publish just moves the task and the link
   travels with it.  Plan files remain git-tracked; the local task is the
   only private part.
