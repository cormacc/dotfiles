* Context

The two existing skills =org-memory= and =plan= form a single coherent
system: =org-memory= owns the durable file protocol, =plan= owns the
section conventions of the change-record artefact, and they reference
each other.  Their current names don't reflect the relationship: the
asymmetric pair (one prefixed with =org-=, the other not) reads like
two unrelated skills, and =org-memory= is an indirect name for what is
fundamentally a task-management system.

This plan adopts *Variant B* from the proposal discussion: rename both
skills to share the =org-= prefix and let =org-memory= survive only as
an informal umbrella term in prose.  The =tasks= extension is *not*
renamed: extension naming optimises for user-facing concision (the
=/tasks= command, the =SPC t t= keybinding, and the \"Tasks\" widget
label all benefit from the short word), while skill naming optimises
for symmetry and namespace clarity in the agent's skill registry.

** Design decisions

- *Skill rename, not extension rename* :: Skills and extensions serve
  different audiences (LLM vs human user) with different
  cost/benefit profiles for verbose names.  The skill rename is a
  documentation-layer change with low blast radius.  The extension
  rename would touch slash commands, keybindings, and the deployed
  filesystem layout in =~/.pi/agent/extensions/= for negligible gain.

- *=org-memory= survives as prose* :: The umbrella concept (\"the
  org-memory protocol\") still has a place in cross-references and
  descriptions, but no longer names a skill.  Sentences like \"the
  org-memory system, implemented by the =org-tasks= and =org-plan=
  skills\" are encouraged.

- *Use =git mv= for directory renames* :: Preserves file history so
  =git log --follow= continues to work on the moved =SKILL.md= files.

- *Description fields are unchanged* :: The =description:= frontmatter
  field is what the LLM matches against when picking a skill.  The
  underlying behaviour didn't change, only the name, so descriptions
  stay intact.  (One small tweak: the description text may incidentally
  mention =org-memory= or =plan=; those mentions are updated to refer
  to the new names where they appear as skill identifiers, not where
  they refer to the umbrella concept.)

- *No backwards-compatibility shim* :: Pre-release; no need to keep
  redirects from old names.  A clean rename is fine.
