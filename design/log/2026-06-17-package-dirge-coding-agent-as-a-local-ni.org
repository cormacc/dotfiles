* Intent

Add =dirge= (upstream =dirge-code/dirge=, a fast pure-Rust coding agent) as a
local nix package in this flake. Ship a from-source build as the default and
installed package, plus a parallel prebuilt-binary package, both cross-platform
(x86_64-linux + aarch64-darwin), following the existing =limux= / =limux-src=
dual-package precedent.

* Summary

*Level:* production · *Tests:* smoke · *Docs:* inline

Two new auto-exposed local packages:
- =pkgs/dirge/= — =rustPlatform.buildRustPackage= from the tagged GitHub source,
  batteries-included default features, pinned via the vendored =Cargo.lock=.
  This is the default and the one installed on Linux workstations.
- =pkgs/dirge-bin/= — prebuilt release tarball per platform (=fetchurl= +
  =autoPatchelfHook=), mirroring the =pkgs/limux/= pattern.

No =flake.nix= / =pkgs/overlay.nix= edits — the overlay auto-exposes any
=pkgs/<name>/default.nix= as =pkgs.<name>= and =packages.<system>.<name>=.

** Scope
*** In scope
- =pkgs/dirge/default.nix= source build (crate =dirge-agent= v0.7.5, binary =dirge=).
- =pkgs/dirge-bin/default.nix= prebuilt binary build.
- Both declare =meta.platforms= = x86_64-linux + aarch64-darwin.
- Install =pkgs.dirge= via =home.packages= in =dev/dev-linux.nix=.
- Build + runtime-smoke verification of both packages.
*** Out of scope
- Upstreaming to nixpkgs.
- aarch64-linux / x86_64-darwin / Windows targets (no current host needs them).
- macOS install wiring (no =dev-darwin= module exists yet; =dirge-bin= / source
  build remain buildable on demand there).
- Trimmed Cargo feature sets, dirge runtime config, shell-plugin integration.

** Acceptance
*** Core functionality
- =nix build .#dirge --impure= succeeds on x86_64-linux.
- The built binary runs and =dirge --version= reports version 0.7.5
  (substring match — output is e.g. =dirge 0.7.5=, do not exact-match).
- =pkgs/dirge= uses =rustPlatform.buildRustPackage= with =cargoLock.lockFile=
  pinned from the upstream =Cargo.lock=.
- Source build uses the batteries-included default feature set (no =--no-default-features=).
- Build uses a Rust >=1.85 toolchain (edition 2024 / MSRV satisfied).
- =nix build .#dirge-bin --impure= succeeds on x86_64-linux.
- =dirge-bin= selects the correct release artifact per platform via =stdenv.hostPlatform=.
- =dirge-bin= Linux binary is patched via =autoPatchelfHook= and runs.
- =pkgs.dirge= is present in =home.packages= in =dev/dev-linux.nix=.
*** Edge cases
- Both derivations declare =meta.platforms= covering x86_64-linux + aarch64-darwin.
- Both carry =meta= with =description=, =homepage=, =license = lib.licenses.gpl3Only=,
  =mainProgram = "dirge"=.
*** Anti-criteria
- Must not edit =flake.nix= or =pkgs/overlay.nix=.
- Must not add OpenSSL / system-TLS build inputs (TLS is rustls, no system
  OpenSSL is linked). NB: =cmake= *is* required (see Decisions) — it is a
  legitimate =nativeBuildInput=, not an anti-criterion.
- Must not install a binary other than =dirge= (the crate may build extra bins
  such as =dirge-microvm-runner=, gated behind a non-default feature).

** Decisions
- Chose source-build (=rustPlatform.buildRustPackage=) as the default :: no
  *system* library dependency to splice in — TLS is rustls (no system OpenSSL),
  so the build is self-contained and cleanly cross-platform from one
  expression, avoiding the foreign-glibc-binary closure problem behind
  =limux='s NVIDIA/Wayland breakage. =cargoLock.lockFile= pins from the
  vendored lock, so no separate =cargoHash= to chase.
  Build-tool caveat (corrected post-review): the vendored =Cargo.lock= resolves
  =rustls 0.23= to its default =aws-lc-rs= backend, so =aws-lc-sys= compiles via
  =cmake= + =cc=, and default features pull =janetrs=/=evil_janet= plus ~12
  tree-sitter grammars that compile C via =cc=. =cmake= is therefore a required
  =nativeBuildInput=; =cc=/stdenv covers the rest. This is a build-time
  toolchain need, not a runtime/system-library dep — the cross-platform and
  closure-cleanliness rationale stands.
  Rejected:
  - Prebuilt-only (=fetchurl= + =autoPatchelfHook=): faster build, but needs a
    per-platform src/hash matrix and re-points a foreign binary at the closure.
    Kept as the secondary =dirge-bin= package, not the default.
  - Trimmed Cargo feature set: leaner closure but a feature list to maintain;
    deferred. Batteries-included matches upstream =cargo install dirge-agent=.
- Provide both packages :: mirrors the =limux= / =limux-src= precedent; gives a
  fast fallback if a future source build breaks or compile time becomes painful.

** Risks
- *cargoHash / rebuild churn on version bumps* — accepted; vendored
  =Cargo.lock= keeps it to a source-hash + rebuild rather than a vendor-hash hunt.
- *Heavy dep tree → multi-minute cold compile* — accepted as the cost of a
  reproducible closure; =dirge-bin= exists for the fast path.
